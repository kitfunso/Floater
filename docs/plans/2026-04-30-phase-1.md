- `AUTO_PAY_RULES: readonly Rule[]` (5 rows: amount < £5k; reliable >=3 on-time; distressScore < 0.3; cash floor preserved across horizon; falls inside MAX_STRETCH window).

**Verify:** `pnpm tsx -e "..."` inline test: `discountAPR(0.02, 10, 30)` returns ~0.372. `classify` of a £30k new-vendor invoice returns `'flagged'`. `AUTO_PAY_RULES.length === 5`.

## Step 6: Optimiser core (greedy + cash-floor + simulate helper + baseline)
**Files:** `lib/optimiser.ts`, `lib/__tests__/optimiser.test.ts`
**What:** Two exports:
- `simulate(entries: ScheduleEntry[], forecast: Forecast): { breachDay: string | null; minBuffer: number }` — pure function used by both the optimiser itself and `agents/cash-impact.ts`.
- `optimise({ invoices, vendors, forecast, distressScores, forceNaive? }): Schedule` — greedy in due-date order. If `forceNaive`, every entry is pay-on-due (used to compute baseline). Otherwise: propose pay-early if `discountAPR > COST_OF_CAPITAL` AND `simulate(...).breachDay === null` after this entry AND `distressScore < 0.3`. Else pay-on-due. If `distressScore > 0.5`, defer to `min(dueDate + 7d, dueDate + MAX_STRETCH_DAYS)` and emit escalation. Returns `Schedule { scheduleId, entries, escalations, totalSaving, breachesAvoidedVsBaseline }`. The route handler (Step 8) computes baseline + diff; this module accepts a `baselineBreaches` input or computes both.
**Verify:** Inline tests: (a) 2/10 net 30, reliable, low distress -> `payDate = issuedDate + 10d`, `reason = 'auto-discount'`. (b) Same with tight cash forecast -> `payDate = dueDate`, no breach. (c) `distressScore = 0.7` -> `payDate <= dueDate + 14d`, escalation present. (d) `forceNaive=true` against the seeded data -> `breachesAvoidedVsBaseline >= 1` once compared to non-naive run. (e) Schedule returned by non-naive run never has `simulate(...).breachDay !== null`.
**Commit:** `feat: optimiser + simulate helper + naive-baseline mode`

## Step 8: API routes (Specter pre-fetch + scheduleId persistence)
**Files:** `app/api/optimise/route.ts`, `app/api/investigate/route.ts`, `app/api/decide/route.ts`, `app/api/execute/route.ts`, `lib/ulid.ts` (or use `crypto.randomUUID()`), `runs/.gitkeep`
**What:**
- `/api/optimise`: read `data/*.json`; `Promise.all` over `getDistressScore(vendor.specterId)` for every vendor; run `optimise({ ..., forceNaive: true })` for baseline; run `optimise({ ..., forceNaive: false })` for full; diff breach counts; mint `scheduleId`; write `runs/<scheduleId>.pending.json`; return Schedule.
- `/api/investigate`: Zod-parse `{ scheduleId, invoiceId }`; read `runs/<scheduleId>.pending.json` to recover the pre-fetched `distressScore`; call `lib/cursor.ts::fanOut`; return `{ verdicts }`.
- `/api/decide`: Zod-parse `{ scheduleId, invoiceId, verdict, reason }`; append to `runs/<scheduleId>.decisions.jsonl`.
- `/api/execute`: Zod-parse `{ scheduleId }`; read pending + decisions; write `runs/<scheduleId>.executed.json`; idempotent.
**Verify:** `curl -X POST localhost:3000/api/optimise -d '{}'` returns Schedule with `scheduleId`, `breachesAvoidedVsBaseline >= 1`, `entries[].distressScore` populated. `runs/<id>.pending.json` exists. `/api/investigate` returns 3 verdicts. `/api/execute` produces an executed log.
**Commit:** `feat: api routes with specter pre-fetch + scheduleId persistence`
