## Step 5: NPV math + policy module
**Files:** `lib/npv.ts`, `lib/policy.ts`
**What:** `npv.ts` exports `discountAPR(discountPct, discountDays, netDays)` and `shouldTakeDiscount(apr, costOfCapital)`. `policy.ts` exports `AUTO_PAY_RULES` and `ESCALATE_RULES` as data + a `classify(invoice, vendor, distressScore, cashImpact): 'auto' | 'flagged'` function. Cost of capital hardcoded at 0.08.
**Verify:** Inline test: `discountAPR(0.02, 10, 30)` returns ~0.372. `classify` of a £30k new-vendor invoice returns `'flagged'`.
**Commit:** `feat: discount apr math + auto-pay policy classifier`

## Step 6: Optimiser core (greedy + cash-floor)
**Files:** `lib/optimiser.ts`, `lib/__tests__/optimiser.test.ts` (one inline test file via `tsx`)
**What:** `optimise({ invoices, vendors, forecast, distressScores }): Schedule`. Greedy: for each invoice in due-date order, propose pay-early if `discountAPR > 0.08` AND simulated cash never breaches floor in 60d AND `distress < 0.3`. Else pay-on-due. If `distress > 0.5`, defer 7d past due. Returns `Schedule { entries, escalations, totalSaving, floorBreaches }`. Escalations come from `policy.classify`.
**Verify:** `pnpm tsx lib/__tests__/optimiser.test.ts` runs three cases: (a) 2/10 net 30 vendor with reliable history & low distress -> entry has `payDate = issuedDate + 10d`, `reason = 'auto-discount'`. (b) Same vendor but cash forecast tight -> entry has `payDate = dueDate`, no breach. (c) High-distress vendor -> entry has `payDate = dueDate + 7d`, escalation present.
**Commit:** `feat: greedy optimiser with cash-floor + vendor-risk gates`

## Step 7: Cursor SDK fan-out + sub-agents
**Files:** `lib/cursor.ts`, `agents/vendor-health.ts`, `agents/cash-impact.ts`, `agents/discount-npv.ts`
**What:** `lib/cursor.ts` exports `fanOut(invoice, vendor, forecast): Promise<Verdict[]>` that runs the three agent files concurrently via `Promise.all` using `@cursor/sdk` (local runtime). Each agent returns `Verdict { agent, recommendation, rationale, score }`. `vendor-health` calls Specter via `lib/specter.ts`. `cash-impact` calls `optimiser.simulate` (extract a pure helper). `discount-npv` calls `npv.ts`. Add an in-memory `Map<invoiceId, Verdict[]>` cache keyed for the session.
**Verify:** `pnpm tsx -e "import('./lib/cursor').then(m=>m.fanOut(...))"` for one invoice returns 3 verdicts in <3s. Hit it twice; second call <50ms (cached).
**Commit:** `feat: cursor sdk parallel sub-agent fan-out with session cache`
